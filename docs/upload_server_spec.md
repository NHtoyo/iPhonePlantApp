# PC側データアップロードサーバー詳細仕様書 (upload_server.py)

このドキュメントでは、iPhoneアプリから送信されるセッションデータ（RGB画像・深度画像・transforms.json等）を受信し展開するPC側のFlaskサーバー（`upload_server.py`）の内部挙動および仕様を解説します。

---

## 1. サーバーの概要と動作要件

* **実装フレームワーク**: Flask (Python)
* **リッスンアドレス**: `0.0.0.0:5000` (同一LAN内のすべての機器からアクセス可能)
* **データ受信・保存先**: `D:\tomato_collection\トマト動画` (スクリプト内 `SAVE_DIR` 変数で指定)

---

## 2. 主要機能とAPI仕様

サーバーは以下の2つの主要APIエンドポイントを提供しています。

### ① データアップロード用: `/upload` (POST)

iPhoneアプリが撮影を終了または手動でアップロードを行った際、データを送信するためのメインAPIです。

* **エンドポイント**: `POST http://<サーバーIP>:5000/upload`
* **データフォーマット**: `multipart/form-data`
  * パラメータ名: `file`
  * ファイル形式: `tar` アーカイブ (例: `nerf_dataset_2026-06-01_14-25-57.tar`)

#### 処理の流れ:
1. **リクエスト検証**: 送信データに `file` パラメータが含まれているか、ファイル名が空でないかを確認します（満たさない場合は `400 Bad Request`）。
2. **メモリ上での受信**: 受信したtarファイルのバイナリデータをメモリ上のバッファ (`io.BytesIO`) に直接読み込みます。これによりディスクへの余分なI/Oを抑えています。
3. **展開先フォルダ名の決定**: 送信されたtarファイルの拡張子（`.tar`）を除いた名前を展開先フォルダ名と決定します。
   * (例: `nerf_dataset_xxx.tar` → フォルダ名 `nerf_dataset_xxx`)
4. **古いデータのクリーンアップ**: すでに同じ名前 of フォルダが存在している場合、後述の `safe_rmtree()` 関数を使って既存フォルダをクリーンアップします。
5. **展開処理**: 新しくフォルダを作成し、メモリ上のtarファイルをPythonの `tarfile` モジュールを用いてそのフォルダ内にすべて展開 (`tar.extractall`) します。
6. **レスポンス**: 正常完了時は `200 Upload successful`、例外発生時は `500 Extraction failed` を返します。

---

### ② フォルダ名変更の同期用: `/rename` (POST)

iPhoneアプリ側でセッションの管理名（フォルダ名）を変更した際に、PCサーバー側の展開済みフォルダの名前もリアルタイムに同期するためのAPIです。

* **エンドポイント**: `POST http://<サーバーIP>:5000/rename`
* **データフォーマット**: `application/json`
* **JSONリクエストボディ**:
  ```json
  {
    "old_name": "変更前のフォルダ名",
    "new_name": "変更後のフォルダ名"
  }
  ```

#### 処理の流れ:
1. **リクエスト検証**: JSONボディ内に `old_name` および `new_name` が含まれているか確認します（ない場合は `400 Bad Request`）。
2. **存在確認**: PC側の `SAVE_DIR` 内に `old_name` で指定されたフォルダが実際に存在するか確認します。存在しない場合はログに警告を出力し、`404 Not Found` を返します。
3. **リネーム処理**: `os.rename()` を用いてフォルダ名を `new_name` に書き換えます。
4. **レスポンス**: 成功時は `200 Rename successful`、エラー発生時は `500` エラーを返します。

## 3. Windows環境に配慮した設計

### Windowsのファイルロック対策 (`safe_rmtree`)
Windows環境では、エクスプローラーでフォルダを開いていたり、他のプロセス（学習用スクリプトなど）がファイルを参照していると、`shutil.rmtree` によるフォルダ削除がエラーで失敗する性質があります。

この対策として、`upload_server.py` では独自の `safe_rmtree()` 関数を実装しています。

* **再試行処理**: 削除に失敗した場合、0.5秒のウェイトを挟んで最大5回まで削除を再試行します。
* **リネーム退避**: 5回再試行してもファイルロックが解除されなかった場合、削除を諦めるのではなく、フォルダ名を `<元のフォルダ名>_deleted_<タイムスタンプ>` にリネームして退避します。
  これにより、アップロード処理自体をエラーにすることなく完了させ、ロックが外れたタイミングで後から手動削除できるように工夫されています。

## 4. サーバーの実行・管理方法

### 起動用バッチファイル
PCのデスクトップ上にある `run_server.bat` から起動できます。

```bat
@echo off
title iPhonePlantApp Upload Server
...
cd /d "%~dp0"
python upload_server.py
```
* **起動手順**: `run_server.bat` をダブルクリックするだけです。
* **依存ライブラリ**: `Flask` が必要です。起動時にエラーになる場合は、コマンドラインで `pip install flask` を実行してください。
